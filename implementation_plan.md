# SleepSense Mobile — Implementation Plan

This document outlines the implementation plan for the **SleepSense Mobile Application**, a cross-platform React Native frontend built with Expo. The mobile app interfaces with the existing FastAPI backend to collect daytime behavioral signals passively, perform daily sleep predictions, and display actionable advice.

---

## Goal Description
Build a beautiful, modern, dark-themed native mobile application (iOS & Android) that:
1. Performs secure, token-based user login & registration.
2. Displays today's predicted sleep quality score via an interactive dashboard widget.
3. Renders personalized sleep advice recommendations driven by SHAP attribution.
4. Plots historical sleep trends (predictions vs actuals).
5. Passively aggregates daytime behavioral metrics (steps, screen time, phone unlocks) in the background and uploads them to the FastAPI server at 10 PM daily.

---

## User Review Required

> [!IMPORTANT]
> **Background Execution Restrictions**: iOS and Android enforce strict power-saving limits on background tasks. While Android's `WorkManager` runs reliably, iOS suspends background scripts if the phone is stationary or in low power mode. In-app passive sensor collection will be implemented via **Expo Background Fetch** and **Task Manager**, but we will include an manual sync button for absolute robustness.

## Proposed Changes & Components

### 1. Authentication Layer (`src/database/storage.js` & `src/screens/LoginScreen.js`)
- Authenticate against `/auth/register` and `/auth/token` on the FastAPI server (running on port 8000).
- Cache the received JWT token securely on-device using `expo-secure-store` (wraps Keychain/Keystore).
- Automatically attach `Authorization: Bearer <token>` to all subsequent Axios request headers.

### 2. Client Dashboard UI (`src/screens/HomeScreen.js`)
- Design a premium dark-themed interface matching the Streamlit dashboard aesthetics.
- Incorporate a custom circular gauge using `react-native-svg` to display the sleep score (0.0 to 3.0) and rounded rating label (Very Good, Fairly Good, etc.).
- Render progress rings for daytime metrics (e.g. step goal, screen time session counts).

### 3. Sleep Coach Advice Panel (`src/screens/AdviceScreen.js`)
- Display the 3 prompt-engineered DistilGPT2 recommendations.
- Integrate the SHAP contribution metrics, visually showcasing which habits (e.g. too many phone unlocks) had the highest negative impact.

### 4. Daytime Telemetry background ETL (`src/sensors/BackgroundETL.js`)
- Set up **Expo Task Manager** to register a background task (`"BACKGROUND_TELEMETRY_SYNC"`).
- Set up **Expo Background Fetch** to trigger the task periodically in the background:
  - On wake-up, fetch step counts from **Expo Pedometer** (CoreMotion/Activity Recognition).
  - Calculate phone pick-up counts using background screen-on listener events.
  - Read location coordinates to compute entropy.
  - Compile the 42-feature JSON object.
  - Sync the payload to `POST /predict` every night.

### 5. Diary & EMA Survey Input (`src/screens/DiaryScreen.js`)
- Provide user-friendly sliders to rate daily stress, happy mood, tiredness, and study hours.
- Provide a text notepad for writing a short daily note, which will be processed using NLP embeddings.

1. **Run in Expo Go**: Scan the QR code generated by `npx expo start` on a physical device.
2. **Verify Auth**: Register a new user, log in, restart the app, and verify the user remains authenticated.
3. **Run Predictions**: Submit a daily journal note and sliders, verify the FastAPI server logs show the request, and check that the mobile screen updates with the 3 custom recommendations.
4. **Check Background Service**: Manually trigger the background telemetry sync task in Expo Go and verify that database records are successfully populated on the backend.
